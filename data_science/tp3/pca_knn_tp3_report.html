def reconstruct(x_i, m, x_bar, PC):
    '''
    reconstruct x_i from X, where x_i is a len 784 vector image.
    '''
    recon = np.zeros((784,))
    for j in range(1, m):
        recon += np.inner((x_i-x_bar), PC[j])*PC[j]+x_bar
    return recon

Reconstruction is done by taking xi, the image we are reconstructing, subtracting the $\hat{x}$, and then applying the innerproduct between the subtraction and the principal components. The principal components are then multiplied and the average image is added.

As I mentioned above, the reconstructed images don't seem to be any different, and it seems to be a problem with how I wrote my reconstruction function. However, the expected result is that adding principal components from m = 1,2,...10 should eventually have m = 10 have the image that most clearly resembles the original image. Choosing a proper $k$ value is all about choosing enough principal components in order to get the closest to the image features you want, while also keeping it low enough that the dimensions are not too high. In a theoretical sense, you want to be able to reduce the dimensions by cutting down on the features that don't really contribute to identifying the image (which corresponds to the later principal components). Which means that the choice of K is about how much you're willing to sacrafice of the image in order to get the dimension down to something manageable.

However, it should be that the more principal components we use, the more accurate the classification should be, since the principal components are adding more features to identify and separate the clusters. In our previous example we talked about 6 and 0, and 7 and 1, which would look similar in some handwriting cases. If we did not have enough principal components during reconstruction, our images would not contain enough features to separate the clusters since 7 and 1 look alike and occupy similar spaces, and the same goes for something like 6 and 0. So we should expect accuracy to increase as we increase m. However, increasing m also leads to increasing dimensions, and that's something we should always keep in mind since in the last TP we saw what increasing dimensions could do to our data.
